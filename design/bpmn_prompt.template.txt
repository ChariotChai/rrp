# 业务需求
{biz_requirement}

例子一：交易数据月度汇总报表
需求描述： 监管机构 [例如：中国人民银行某分行 或 国家金融监督管理总局某部门] 要求本支付机构每月结束后 5 个工作日内，提交上一个月度全平台的交易数据汇总报表。该报表需包含但不限于：总交易笔数、总交易金额（区分人民币及外币，并注明外币币种及折算汇率）、按交易类型分类的明细（如：消费、转账、充值、提现等）交易额占比、按商户类型分类的交易额占比、最大单笔交易金额、交易成功率等关键指标。报表需加盖公司公章，电子版发送至指定邮箱，纸质版快递至指定地址。此报表旨在监测支付机构的业务规模、交易活跃度及潜在风险，确保资金流向符合规定。

例子二：大额及可疑交易监控报表
需求描述： 根据《反洗钱法》及相关支付机构反洗钱规定，监管机构 [例如：国家金融监督管理总局反洗钱部门] 要求本支付机构每日结束后 2 小时内，提交当日发生的大额交易及系统自动标记的可疑交易监控报表。大额交易标准参照《金融机构大额交易和可疑交易报告管理办法》执行（例如：单笔或当日累计人民币 20 万元以上或外币等值 1 万美元以上的交易）。可疑交易指系统根据预设规则（如交易频率异常、交易对手关系复杂、资金来源不明等）识别出的潜在风险交易。该报表需详细列出交易时间、交易双方（商户/用户/钱包）信息、交易金额、交易渠道、交易目的、系统风险评级、以及支付机构初步审核意见（如：已核实、待核实、已上报等）。此报表旨在及时发现并阻断可能涉及洗钱、恐怖融资等违法犯罪活动的资金流动，履行支付机构的反洗钱义务。

平台提供了以下能力节点，每个能力需要提供必要的参数才能使用：

```yaml

- node_type: start
  capability: 起始节点，整个流程中有且仅有一个

- node_type: data_collection
  capability: 在数据库中执行SQL获得结果
  input:
    - name: sql 
      type: string
      comment: 一个完整的query类型的SQL


- node_type: data_review
  capability: 等待用户审核，等待一定时间后如仍未完成则失败
  input:
    - name: wait_time_minute
      type: int
      comment: 等待时间(分钟)

- node_type: send_email
  capability: 向指定地址列表发送邮件
  input: 
    - name: receivers
      type: string 
      comment: 收件人列表
    - name: content
      type: string 
      comment: 邮件内容

- node_type: data_review
  capability: 等待用户审核，等待一定时间后如仍未完成则失败
  input:
    - name: wait_time_minute
      type: int
      comment: 等待时间(分钟)

```
